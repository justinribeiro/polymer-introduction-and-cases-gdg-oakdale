How is this sorcery possible?

Four major API’s within browsers that allow us to build a web component

  1. Custom Elements
  2. Shadow DOM
  3. Templates
  4. HTML Imports

Custom Elements

  • They allow us to define new HTMLElements (think new tag)
  • Simple, easy to read, declarative
  • It actually means something: I know we’re talking about tabs

Templates

We’ve all used them: handlebars, mustache, fuzzy-navel. What if the browser did this for us?

  • Uses the DOM to scaffold, which means no Cross Site Scripting
  • Parsed but doesn’t render, inert until use
  • Fragment: not part of the page

Shadow DOM

Scope me all the things.

  • Sealed components: the page styles and code don’t affect them
  • Right back at you: your components are scoped and won’t leak into the page

HTML Imports

Think of the dependency management hell you’re in.

We can wrap up our dependencies into one component and simply import it.

What’s the browser support look like?

  • Chrome since 36 support all natively
  • Firefox is has support in nightly
  • Opera inheirits from Blink
  • IE is looking into it
  • Safari...who knows

Hummmmfff. Well, we’re going to need to polyfill

Enter Polymer

The pieces

  • platform.js fills in the gaps as the polyfill
  • polymer.js adds some syntactic sugar

Syntactic sugar?

  • Declarative syntax
  • Data Binding
  • Mutation Observation

I’m really cutting back on my sugar intake Justin

  • That’s cool, you don’t have to use the Polymer sugar, you can just use the platform.js polyfill (also used by Mozilla's x-tags)
  • Polymer is Google’s way of thinking: let’s make it easier to work

Walking through boilerplate


<polymer-element name="my-element">
    <script>
        Polymer('my-element', {
            // Fires when an instance of the element is created
            created: function() {},

            // Fires when the element’s initial set of children and siblings are guaranteed to exist
            domReady: function() {},

            // Fires when the "<polymer-element>" has been fully prepared
            ready: function() {},

            // Fires when the element was inserted into the document
            attached: function() {},

            // Fires when the element was removed from the document
            detached: function() {},

            // Fires when an attribute was added, removed, or updated
            attributeChanged: function(attr, oldVal, newVal) {}
        });
    </script>
</polymer-element>

Extending into Hello World


<polymer-element name="hello-world" attributes="who">
    <template>
        <p>Hello <strong>{{who}}</strong> :)</p>
    </template>

    <script>
        Polymer('hello-world', {
            who: 'World'
        });
    </script>
</polymer-element>
